VSCode Java开发

VsCode新版Java插件不再支持JDK 1.8。 想要编译开发Java 1.8程序，可以按照以下操作支持JDK1.8。

环境安装

  1. VSCode (本文VSCode 1.48作为演示）
  2. JDK 1.8 (目前是1.8_271)
  3. JDK 11 (如果只安装JDK 1.8版，VsCode Java插件无法工作）

VSCode插件安装

  1. Java Extension Pack (最新版，目前是0.11.0)

配置

  1. 配置环境变量： JDK_HOME->为JDK 11的路径。（为了让VsCode的Java插件正常工作）
  2. VsCode配置（settings.json），在合适位置增加以下内容：（为了设置工程使用Java 1.8编译）
    "java.configuration.runtimes": [

        {
            "name": "JavaSE-1.8",
            "path": "C:\\Program Files\\Java\\jdk1.8.0_271",
            "default": true
        },]

开发

新建Java工程

  1. 打开VsCode，Ctrl+Shift+p，输入java:create，回车。
  2. 选择No build tools，然后选择创建工程文件夹。
  3. 输入工程名称，并回车，完成。

调试/运行

  1. VsCode 文件管理窗格中，双击App.java打开，等待Java插件运行结束。
  2. 此时右上角有运行和调试的按钮（系统关闭按钮的下方附近）
  3. 因为没有使用build tools，所以三方jar包自行放在lib文件夹中。

发布jar包

  1. 文件浏览窗格的下方，有个JAVA PROJECTS的窗格，鼠标移动到这个窗格上，会浮现几个按钮，其中一个箭头图标的按钮就是导出jar。
